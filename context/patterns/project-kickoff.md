# Project Kickoff — Scope to Engineering

How Pulse transitions an approved client scope into active engineering work.

## Step 1: Build User Stories from Scope

Once the client project scope is approved, break it into user stories in ClickUp:

- **User story** — title + description covering the user-facing outcome
- **Subtasks** under each story — each with:
  - Title
  - Description
  - Assignee
  - Priority flag
  - Due date

These can be sent to ClickUp via MCP and automatically added to the project list.

## Step 2: Engineer Review

- Engineer reads through all user stories and subtasks
- If no questions → they're ready to start
- If questions → set up a call with Jake (or project lead) to walk through and clarify
- Goal: engineer has full context before writing any code

## Step 3: Autonomous Development

- Engineer works through user stories one by one
- **Claude Code is the primary developer** — engineer reviews and guides
- Jake (or project lead) serves as code reviewer
- Engineer has freedom to make decisions based on account context
- Work fast, iterate, ship — speed is the priority

## Key Principles

- Engineers should have enough context to run autonomously
- Claude Code does the coding; humans review
- User stories should be specific enough that an engineer + Claude can execute without constant check-ins
