# AI Transformation Roadmap

How Pulse structures a year-long AI transformation engagement (e.g., SWG, future DCC phases).

## Discovery Phase

Extensive discovery across the entire business. This is not a single-feature build — it's understanding every facet of how the business operates.

### What to map
- **Each software system** they use — what it does, where data lives, how it's structured
- **Existing business processes** — how work flows between people and systems today
- **Business goals** — what the company is trying to achieve, where AI creates the most leverage
- **Data landscape** — what data exists, where it's stored, how it's formatted, what's accessible via API

### The core customer need

These customers want to **talk to all their data sources in one place** — not siloed views per system, but everything integrated to support business-level decisions.

## Implementation Pattern

1. **Understand the data** — inventory every system, API, database, file store
2. **Format the data** — normalize and structure data for AI consumption (ETL pipelines, vector databases)
3. **Process the data** — build retrieval and ranking systems (embeddings, search, filtering)
4. **Translate to readable format** — make data AI-readable so users can chat against it naturally

## Deliverables (typical)

- Unified chat experience across all data sources
- Purpose-built AI tools for specific workflows (e.g., tax one-pagers, RMD agent)
- Process automation for repetitive tasks
- Reports and documentation generated from integrated data
- Ongoing optimization as new use cases emerge

## Timeline

Typically 12 months. Phases roll out incrementally — ship working tools early, expand coverage over time.
