# Client Project Scoping

How Pulse scopes a new client engagement from first contact to approved project.

## Process

1. **Initial meeting** (Sean Miller, CEO) — strategy and discovery call with the customer
   - Learn the business problem at a process level
   - Understand what's happening day-to-day, not just what they think the solution should be
   - Brainstorm how Pulse can build a solution around the real problem

2. **Business process mapping** — understand the workflow before proposing technology
   - Start at the trigger point: what kicks off the process?
   - Walk through each step they currently take
   - Identify edge cases and how they're handled today
   - Document this as user flow charts

3. **Solution design** — translate the business problem into a buildable scope
   - Match the problem to Pulse's stack capabilities (Flask, React, Odoo, AI, etc.)
   - Identify integrations needed (CRMs, ERPs, APIs, data sources)
   - Define deliverables and phases

4. **Project scope approval** — client signs off on the scope before engineering begins

## Key Principle

Understand the business process first, then design the technology. Never lead with a solution before understanding the problem at a process level.
