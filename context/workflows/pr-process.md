# PR and Git Workflow

## Git Flow

1. **Feature branches** — all development happens here
2. **Push to remote** — publish feature branch
3. **PR to staging** — merge feature branch into staging
4. **Team testing** — everyone tests on staging
5. **PR to main** — close the staging PR once validated

## Guidelines

- Engineers are encouraged to work fast and iterate; speed over perfection
- Code review mindset: "we want to be code reviewers, not coders" — AI does the coding, humans review
- Use the `/review` skill before opening a PR to catch structural issues
- Use the `/ship` skill to automate the release workflow
